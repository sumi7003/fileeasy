# FileEasy 真实落地进度清单 v1.0

## 1. 文档信息

- 产品：`FileEasy`
- 文档类型：`真实落地进度清单`
- 版本：`v1.0`
- 日期：`2026-05-08`
- 目标：把当前代码现状从“高保真 / demo 视角”切换到“正式产品 / 真实业务 / 端到端验证”视角

## 2. 当前结论

当前 FileEasy 已经明显超过“高保真原型”阶段。

它现在的真实状态是：

- `正式 Web 页面` 已存在
- `Android FileEasy 壳层` 已存在
- `上传后端与文件管理接口` 已存在
- `上传 / 管理核心业务逻辑` 已存在
- `真实设备端到端验证` 仍未闭环
- `高保真视觉` 还没有完整、统一地进入正式产品

一句话总结：

`核心业务链路已实现 70% - 80%，但还没有完成产品收口和真实交付验证。`

## 3. 已真实落地的部分

### 3.1 正式路由

- `/`：上传页
- `/admin/*`：管理页
- `/fileeasy-demo`：原型演示页

说明：

- 主路由已经是正式业务页
- `fileeasy-demo` 现在应只保留为辅助参考，不再作为主开发目标

### 3.2 Web 上传页

已落地：

- 登录前页面可见
- 密码验证后才能上传
- 多文件上传
- 单文件 4GB 限制
- 8MB 分片上传
- 本地任务恢复
- 断网后暂停
- 已存在 uploadId 时续传
- 上传完成回写最终状态

当前判断：

- 这不是 demo 上传页
- 这是已经接真实接口的正式业务页

### 3.3 Web 管理页

已落地：

- 密码登录
- 文件列表
- 按类型筛选
- 搜索
- 预览
- 下载
- 重命名
- 单文件删除
- 批量删除

当前判断：

- 管理页已经具备正式业务骨架
- 剩下主要是产品打磨与端到端验证，不是从零实现

### 3.4 Android FileEasy 壳层

已落地：

- FileEasy flavor 单独入口
- 启动时进入 FileEasy 壳层
- 本地服务拉起
- LAN 地址解析

- 管理后台入口
- 修改密码
- Web 资源拷贝初始化

当前判断：

- Android 侧不是空壳
- 但首页视觉和正式高保真还没有完全统一

## 4. 不应再继续投入的方向

以下内容不应该再作为主线继续深挖：

- `fileeasy-demo` 的场景切换器
- prototype 常量继续细拆
- prototype 容器继续抽象
- 为 demo 服务的 scene / layout / config 再优化

说明：

- 这些代码可以保留
- 但它们不应该再吃掉正式产品落地的时间

## 5. 当前真实堵点

### 5.1 正式页面主线还不够干净

当前问题：

- 上传页正式实现已迁回 `src/fileeasy/pages/UploadPage.tsx`
- 旧 `src/pages/FileEasyUploadPage.tsx` 目前仅保留兼容桥接
- 但样式与少量历史引用仍未完全清走

影响：

- 正式主线已经开始变干净
- 但仍需要继续清理旧 `pages` 与 prototype 依赖，避免历史结构继续反向牵制正式页

### 5.2 高保真没有完整进入正式产品

当前问题：

- Web 端有比较明确的高保真方向
- Android 首页仍然主要是功能型 Compose 壳层
- 与前期高保真设计稿尚未完全统一

影响：

- 产品观感不一致
- 评审时会感觉“业务有了，但产品还没真正成型”

### 5.3 真实设备端到端验证缺失

当前缺口：

- 真机扫码进入上传页
- 真机上传完成后在 `/admin` 可见
- Wi-Fi / 热点访问
- 断网恢复
- 页面重进后续传
- 重启恢复

影响：

- 现在还不能说“可交付”
- 只能说“代码链路大体具备”

## 6. 接下来应做的顺序

### Phase 1：收主线

目标：

- 让正式页面真正回到 `src/fileeasy/pages/*`
- 让旧 `src/pages/*` 只保留兼容桥接或彻底退出主线

优先项：

1. `UploadPage` 正式实现归位
2. `AdminPage` 保持为正式主线
3. `ApkHomePage` 与 Android 首页职责对齐

### Phase 2：收产品

目标：

- 把高保真真正落进正式页面与正式壳层

优先项：

1. APK 首页视觉重做为正式高保真
2. 上传页视觉与交互统一
3. 管理页工作台视觉统一

### Phase 3：跑真实链路

目标：

- 做真实端到端验证

优先项：

1. 真机访问上传页
2. 真机上传文件
3. `/admin` 查看文件
4. 预览 / 重命名 / 下载 / 删除
5. 断网恢复与续传
6. Wi-Fi / 热点 / 重启验证

直接执行以下三步：

1. 清点哪些 prototype 代码退出主线

3. 开始真机端到端联调
