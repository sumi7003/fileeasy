# FileEasy PRD v1.0

## 1. 文档信息

- 产品名：`FileEasy`
- 版本：`v1.0`
- 状态：`定稿`
- 产品类型：Android 局域网文件接收服务 APK

## 2. 产品目标

FileEasy 的目标是把一台 Android 手机、平板或盒子变成局域网内可访问的文件接收节点，让手机、电脑等非 APK 设备通过扫码或输入地址直接上传文件到该 Android 设备。

## 3. 产品边界

- FileEasy 是纯服务端 APK，不带播放器功能。
- 仅服务于局域网场景，支持 Wi-Fi 和热点。
- 不做自动发现，不做公网访问。
- 远程用户唯一页面是上传页 `/`。
- v1 不提供远程管理页 `/admin`。
- 远程文件列表、预览、重命名、删除、批量删除、下载均不属于 v1 范围。
- 后续如需文件管理能力，应作为 v1.1 或独立需求重新设计。

## 4. 用户角色

### 4.1 管理员

- 安装并启动 APK
- 查看二维码和上传地址
- 修改系统密码
- 在 APK 本机查看服务状态并维护上传入口

### 4.2 上传用户

- 扫码或输入地址进入上传页 `/`
- 登录后上传文件

当前版本中，管理员与上传用户共用一套密码。

## 5. 认证规则

- 安装后系统带有默认密码。
- 默认密码支持修改。
- 上传页使用该密码登录。
- 修改密码仅允许通过 APK 本机入口完成。
- 登录态保持 7 天。
- 用户进入上传页后先看到页面，再进行密码验证。

## 6. 核心流程

1. 管理员安装并打开 FileEasy APK。
2. APK 首页显示上传二维码和上传地址。
3. 上传用户扫码进入上传页。
4. 上传用户输入密码并登录。
5. 上传用户上传单文件或多文件。
6. 文件保存到 Android 本地存储。

## 7. APK 首页需求

首页采用极简方案，仅展示：

- 服务状态
- 上传页二维码
- 上传地址
- 修改密码入口

首页不展示：

- 监控面板
- 日志面板
- 专业系统指标
- 复杂技术说明

## 8. 上传能力

- 支持单文件上传
- 支持多文件上传
- 支持断点续传
- 单文件最大支持 `4GB`
- 分片大小固定 `8MB`
- 未完成上传任务保留 `24 小时`
- `24 小时`后清理未完成上传状态

## 9. 文件命名规则

- 同名文件上传时自动重命名，如 `文件(1).ext`
- v1 不提供远程重命名能力。

## 10. 存储规则

- 文件默认永久保存
- 不做自动过期删除
- 空间不足时：
  - 禁止发起新的上传
  - 上传过程中空间耗尽则上传失败
  - 页面提示管理员清理空间

## 11. 文件类型范围

第一版支持以下扩展名：

- 文档：`pdf doc docx xls xlsx ppt pptx txt`
- 图片：`jpg jpeg png gif webp`
- 视频：`mp4 mov`
- 音频：`mp3 wav m4a`
- 压缩文件：`zip`

第一版不补充更多类型范围。

## 12. 远程文件管理边界

v1 不提供远程文件管理能力，包括：

- `/admin` 管理页
- 远程文件列表
- 远程在线预览
- 远程文件重命名
- 远程文件删除
- 远程批量删除
- 远程文件下载

上述能力不作为 v1 验收项。后续如需补充，应先更新 PRD 和设计文档，再作为 v1.1 或独立需求派发。

## 13. 上传页 API 边界

远程 Web 表面仅保留上传页所需 API：

- 服务可达性与首页摘要
- 登录与退出
- 上传初始化
- `8MB` 分片上传
- 上传状态查询
- 上传完成
- 未完成上传取消

不暴露远程文件管理 API，例如 `/api/v1/files`、预览、下载、重命名、删除、批量删除等接口。

## 14. 日志要求

记录以下操作日志：

- 上传

每条日志至少包含：

- 操作时间
- 操作 IP
- 文件名
- 操作类型

## 15. 运行要求

- APK 支持开机自启
- APK 以前台服务方式常驻
- 局域网内可被其他设备访问
- 支持热点网络场景

## 16. 非功能要求

- 上传页需适配手机浏览器与桌面浏览器
- 大文件上传过程中，浏览器刷新或中断后可续传
- 页面需有明确错误提示和状态反馈
- 不因播放器逻辑影响 FileEasy 功能

## 17. 第一版验收标准

满足以下条件即视为 v1 可交付：

- 可上传单个 `4GB` 文件
- 可进行多文件上传
- 分片大小固定为 `8MB`
- 上传中断或刷新后可继续上传
- 扫码可进入上传页 `/`
- 上传页密码登录可用
- 不支持类型和超过 `4GB` 的文件被明确拒绝
- 空间不足时给出明确提示并阻止或终止上传
- 支持 Wi-Fi 和热点局域网访问
- 支持网络切换后的地址与二维码刷新
- 支持开机自启和前台服务常驻
- 上传操作具备基础日志记录

## 18. 当前明确不做

- 播放器功能
- 自动设备发现
- 公网访问
- 自动清理旧文件
- `/admin` 远程管理页
- 远程文件列表
- 远程在线预览
- 远程文件重命名
- 远程文件删除
- 远程批量删除
- 远程文件下载
- 批量下载
- 复杂系统监控首页

## 19. 管理要求

- 设计文档不得超出本 PRD 范围。
- AI 开发任务不得自行扩需求。
- 若需求变更，先更新 PRD，再更新设计文档，再重新派发任务。
