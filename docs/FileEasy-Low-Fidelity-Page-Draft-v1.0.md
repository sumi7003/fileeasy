## 2. 文档目标

本稿用于把线框说明进一步转成“低保真页面草图描述”，明确每一屏的视觉重心、区块比例、阅读动线、控件密度和页面气氛，方便后续 UI 出低保真稿、高保真稿和页面实现。

本稿仍然不进入高保真视觉细节，不定义具体像素值与视觉样式，只定义页面草图层级和构图方式。

## 3. 使用方法

阅读本稿时，可把每一页理解为：

- 页面先让用户看到什么
- 页面中部由什么占主导
- 页面底部承接什么动作
- 异常状态替换掉哪个区块
- 手机端与桌面端在结构上怎么变化

## 5. `LF-A01` APK 首页

### 5.1 页面定位

这是一张“设备端服务入口页”，不是后台，也不是控制台。视觉上应像一个安静、明确、可以被远距离理解的服务主页。

### 5.2 首屏印象

用户打开后，第一眼应该看到：

1. 这是 `FileEasy`
2. 服务是否可用
3. 二维码在哪里

### 5.3 草图构图

页面采用单列纵向构图，整体像“三段式海报”：

- 上段：产品名 + 状态
- 中段：二维码大卡片
- 下段：管理页与修改密码入口

二维码卡片占全页视觉注意力的 `50%` 左右，其他内容都应让位于它。

### 5.4 草图说明

```text
----------------------------------------------------
 顶部留白

 FileEasy
 把这台设备作为局域网文件接收节点

 [服务已开启]

 ┌──────────────────────────────────────────────┐
 │ 上传二维码                                   │
 │                                              │
 │                [ 大号二维码 ]                │
 │                                              │
 │ 上传页地址                                   │
 │ http://192.168.1.23:3000/                    │
 └──────────────────────────────────────────────┘

 ┌──────────────────────────────────────────────┐
 │ 进入管理页                                   │
 │ 查看和管理已上传文件                         │
 └──────────────────────────────────────────────┘

 ┌──────────────────────────────────────────────┐
 │ 修改密码                                     │
 │ 更新上传页与管理页共用密码                   │
 └──────────────────────────────────────────────┘

 底部留白
----------------------------------------------------
```

### 5.5 视觉重心

- 最大块：二维码卡片
- 第二重心：服务状态
- 第三重心：管理页入口

### 5.6 无地址状态

无可用局域网地址时，中部大卡片不再放二维码，改为一张“不可用提示卡”。

```text
----------------------------------------------------
 FileEasy
 [未检测到可用局域网地址]

 ┌──────────────────────────────────────────────┐
 │ 当前无法生成上传二维码                       │
 │ 请确认设备已连接 Wi-Fi 或热点                │
 └──────────────────────────────────────────────┘

 [进入管理页]
 [修改密码]
----------------------------------------------------
```

### 5.7 适配建议

- 手机：二维码卡尽量完整露出在首屏
- 平板：可以放大中部卡片并增加留白
- 电视盒子：标题和辅助文案缩短，优先二维码和地址可见

## 6. `LF-W01` 扫码上传页登录前

### 6.1 页面定位

这是“先理解用途，再进行验证”的上传页，不应做成强拦截登录页。

### 6.2 首屏印象

用户打开后，应该先理解：

1. 这是文件上传页
2. 支持多文件和断点续传
3. 需要密码后才能开始

### 6.3 草图构图

采用“背景是上传页，前景是登录卡”的双层草图结构：

- 背景层：真实上传页面骨架，但不可操作
- 前景层：登录卡片，位于页面中上部

### 6.4 草图说明

```text
----------------------------------------------------
 FileEasy 文件上传
 上传到当前局域网设备

 ┌──────────────────────────────────────────────┐
 │ 选择文件                                     │
 │ 支持单文件 / 多文件上传                      │
 │ 单文件最大 4GB，支持断点续传                 │
 └──────────────────────────────────────────────┘

            ┌──────────────────────────┐
            │ 输入密码后开始上传文件   │
            │ [ 密码输入框         ]   │
            │ [ 登录 ]                 │
            └──────────────────────────┘

 上传任务将在登录后显示
----------------------------------------------------
```

### 6.5 视觉重心

- 第一重心：登录卡
- 第二重心：上传能力卡
- 第三重心：标题

### 6.6 错误态

密码错误时，不跳全页错误，只在登录卡内部加一行错误提示。

```text
┌──────────────────────────┐
│ 输入密码后开始上传文件   │
│ [ 密码输入框         ]   │
│ 密码错误，请重新输入     │
│ [ 登录 ]                 │
└──────────────────────────┘
```

## 7. `LF-W02` 扫码上传页登录后

### 7.1 页面定位

这是一个“上传操作 + 上传进度”双核心页面。用户选完文件后，不应该迷路，页面始终要告诉他文件正在发生什么。

### 7.2 手机端构图

手机端采用上下堆叠：

- 顶部：标题
- 中部：上传卡
- 下部：任务列表

任务列表是会不断变长的主体区域。

### 7.3 手机端草图

```text
----------------------------------------------------
 FileEasy 文件上传
 已连接到当前设备

 ┌──────────────────────────────────────────────┐
 │ [ 选择文件 ]                                 │
 │ 支持单文件 / 多文件上传                      │
 │ 单文件最大 4GB，支持断点续传                 │
 └──────────────────────────────────────────────┘

 文件 A.pdf                               25%
 [███████-----------------------------]
 48 MB | 上传中

 文件 B.zip                           等待上传
 [-----------------------------------]
 1.2 GB

 文件 C.mp4                           上传完成
 [██████████████████████████████████]
 860 MB
----------------------------------------------------
```

### 7.4 桌面端构图

桌面端采用左右双栏：

- 左栏较窄：上传操作
- 右栏较宽：任务列表

这样能让用户一边继续加文件，一边持续观察进度。

### 7.5 桌面端草图

```text
--------------------------------------------------------------
 FileEasy 文件上传

 ┌───────────────────────┐  ┌───────────────────────────────┐
 │ [ 选择文件 ]          │  │ 文件 A.pdf               25%  │
 │ 支持单/多文件上传     │  │ [进度条]                    │
 │ 单文件最大 4GB        │  │ 48 MB | 上传中              │
 │ 支持断点续传          │  │                             │
 │                       │  │ 文件 B.zip           等待上传 │
 │                       │  │ [进度条]                    │
 └───────────────────────┘  └───────────────────────────────┘
--------------------------------------------------------------
```

### 7.6 视觉重心

- 手机端：先按钮，后进度
- 桌面端：右侧任务列表略强于左侧上传区

## 8. `LF-W03` 上传进行中与完成态

### 8.1 页面定位

这不是新页面，而是上传页里的核心状态草图。重点是任务项本身要足够稳定、统一、好扫读。

### 8.2 单个任务草图模板

```text
文件名
[进度条]
辅助信息
```

### 8.3 进行中

```text
宣传视频.mp4                               42%
[████████████--------------------------]
860 MB | 上传中
```

### 8.4 正在恢复

```text
宣传视频.mp4                           正在恢复
[████████████--------------------------]
860 MB | 正在恢复上传任务
```

### 8.5 上传完成

```text
宣传视频.mp4                           上传完成
[████████████████████████████████████]
860 MB
```

### 8.6 上传失败

```text
宣传视频.mp4                           上传失败
[█████████----------------------------]
设备空间不足，请联系管理员清理
```

### 8.7 草图要求

- 每个任务块高度尽量一致
- 文件名优先完整展示
- 失败态不需要更大块，只需更明确文案
- 完成态应收敛，不要做过度庆祝式反馈

## 9. `LF-W04` 上传异常态

### 9.1 页面定位

异常态不应把页面打碎，而应在原页面上插入“明确但克制”的提醒块。

### 9.2 空间不足草图

```text
----------------------------------------------------
 FileEasy 文件上传

 ┌──────────────────────────────────────────────┐
 │ 设备空间不足，请联系管理员清理               │
 │ 当前无法发起新的上传                         │
 └──────────────────────────────────────────────┘

 ┌──────────────────────────────────────────────┐
 │ [ 选择文件 ]  按钮禁用                        │
 └──────────────────────────────────────────────┘

 任务列表仍然可见
----------------------------------------------------
```

### 9.3 网络中断草图

```text
文件 A.pdf
[██████████--------------------------]
网络中断，正在等待恢复
```

### 9.4 续传过期草图

```text
文件 A.pdf
[██████████--------------------------]
上传任务已过期，请重新上传
```

## 10. `LF-W05` 管理页列表

### 10.1 页面定位

管理页不是复杂后台，而是“文件操作页”。重点是快速找到文件，快速进行预览、下载、删除和重命名。

### 10.2 手机端构图

手机端采用“搜索在上，文件卡片在下”的列表流：

- 顶部标题
- 搜索框
- 批量删除入口
- 文件卡片列表

### 10.3 手机端草图

```text
----------------------------------------------------
 文件管理

 [ 搜索文件名                                ]

 [ 批量删除 ]

 ┌──────────────────────────────────────────────┐
 │ 项目方案.pdf                                 │
 │ PDF | 12.4 MB | 2026-05-07 14:20            │
 │ [预览] [下载] [更多]                         │
 └──────────────────────────────────────────────┘

 ┌──────────────────────────────────────────────┐
 │ 宣传视频.mp4                                 │
 │ 视频 | 256 MB | 2026-05-07 12:10            │
 │ [预览] [下载] [更多]                         │
 └──────────────────────────────────────────────┘
----------------------------------------------------
```

### 10.4 桌面端构图

桌面端采用轻表格结构：

- 顶部标题
- 搜索与批量入口同行
- 下方文件表格

### 10.5 桌面端草图

```text
----------------------------------------------------------------
 文件管理

 [ 搜索文件名                      ]            [ 批量删除 ]

 文件名            类型     大小        时间              操作
 --------------------------------------------------------------
 项目方案.pdf      PDF      12.4 MB     2026-05-07 14:20  预览 下载
 宣传视频.mp4      视频     256 MB      2026-05-07 12:10  预览 下载
----------------------------------------------------------------
```

### 10.6 空状态

```text
当前暂无文件
```

### 10.7 搜索无结果

```text
未找到相关文件
```

## 12. `LF-W07` 管理页重命名

### 12.1 页面定位

重命名是一个小而明确的编辑动作，应该使用轻弹窗，不要让用户离开列表上下文。

### 12.2 草图

```text
----------------------------------------------------
 重命名文件                                  [关闭]

 主文件名
 [ 项目方案_终版                    ][ .pdf ]

 [ 取消 ]                              [ 保存 ]
----------------------------------------------------
```

### 12.3 草图要求

- 扩展名单独锁定显示
- 输入框宽度大于扩展名区域
- 保存按钮为主按钮

## 13. `LF-W08` 管理页删除确认

### 13.1 页面定位

删除确认是危险动作确认层，视觉上应更简洁、更聚焦，不出现多余说明。

### 13.2 单文件删除草图

```text
----------------------------------------------------
 确认删除

 删除后无法恢复
 是否删除“项目方案.pdf”

 [ 取消 ]                          [ 确认删除 ]
----------------------------------------------------
```

### 13.3 批量删除草图

```text
----------------------------------------------------
 确认批量删除

 删除后无法恢复
 是否删除已选 12 个文件

 [ 取消 ]                          [ 确认删除 ]
----------------------------------------------------
```

## 14. `LF-W09` 管理页批量删除模式

### 14.1 页面定位

进入批量模式后，整个列表页应暂时切换到“选择状态”，页面不再鼓励单文件逐个操作。

### 14.2 手机端草图

```text
----------------------------------------------------
 已选择 3 个文件                           [退出]

 [ 删除已选文件 ]

 [√] 项目方案.pdf
 PDF | 12.4 MB | 2026-05-07 14:20

 [√] 宣传视频.mp4
 视频 | 256 MB | 2026-05-07 12:10
----------------------------------------------------
```

### 14.3 桌面端草图

```text
----------------------------------------------------------------
 文件管理                                           [退出批量]

 已选择 3 个文件                                 [删除已选文件]

 勾选   文件名           类型     大小        时间
 --------------------------------------------------------------
 [√]   项目方案.pdf      PDF      12.4 MB     2026-05-07 14:20
 [√]   宣传视频.mp4      视频     256 MB      2026-05-07 12:10
----------------------------------------------------------------
```

### 14.4 草图要求

- 批量条必须始终可见
- 删除已选文件是唯一主动作
- 单文件“更多”操作在此模式下弱化或隐藏

## 15. 页面阅读动线总结

3. 页面标题

1. 选择文件
2. 当前上传任务

### 15.3 管理页

视线顺序：

1. 搜索框
2. 文件名
3. 文件操作
4. 批量删除入口
