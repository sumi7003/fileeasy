

## 局域网文件传输功能需求文档

### 1. 功能概述

本功能旨在为 Web 管理页面新增局域网文件传输能力，支持用户在局域网内便捷地上传、分享和下载文件。该功能提供完整的文件生命周期管理，包括上传、下载、权限记录、自动过期清理等核心能力。

**核心价值：**
- 简化局域网内文件共享流程
- 提供安全的文件传输记录追溯
- 自动化文件存储管理，避免垃圾文件堆积
- 支持二维码快速分享，提升用户体验

---

### 1.1 基于现有 Xplay 项目的前置说明（必须遵循）

本需求需要**基于当前 Xplay（Android Host Mode）架构落地**，关键约束如下：

- **原则（不改动既有链路）**：不得改动 Xplay 原有"设备注册/心跳/播放列表下发/素材库上传与播放"等链路与逻辑；本功能以**新增模块/新增接口/新增页面**方式实现，默认不影响现有页面与接口的行为、性能与数据结构。
- **部署形态**：管理后台由主Pad内置本地服务提供（Ktor/Netty），管理页入口为 `http://<主PadIP>:3000/` 或 `http://eastaifile.com:3000/`（需要 DNS/Hosts 配置），API 前缀为 `/api/v1`。
- **入口与导航**：在管理页面"素材库"菜单旁边新增"**文件中转**"菜单项，进入文件传输管理页面。
- **鉴权方式**：管理页有密码登录（`xplay_auth=admin-token` Cookie）。分享下载采用**统一账密鉴权**方式：
  - **下载账户**：`eastai`
  - **下载密码**：`888888`
  - 分享链接格式：`http://<主PadIP>:3000/t/<fileId>` 或 `http://eastaifile.com:3000/t/<fileId>`
  - 访问分享链接时需要输入账密，验证通过后才能下载
- **现有能力复用**：项目已存在"素材库上传（/api/v1/media/upload）"与"APK 分发（/api/v1/update/*）"等上传/下载能力；本需求应复用相同的上传机制与存储目录策略，但**不能**直接复用 `/uploads/{filename}` 作为公开分享入口（否则无法做下载审计与过期控制）。
- **存储位置**：文件落盘在主Pad应用私有目录（`filesDir` 下），空间上限取决于设备磁盘；空间占用与剩余情况在**监控页面**展示（与系统监控整合）。

### 2. 功能需求

#### 2.1 文件上传功能

**需求描述：**
用户可以在管理页面中上传文件到服务器，支持多种文件类型。

**功能要点：**
- 支持单文件和多文件上传
- 支持拖拽上传和点击选择上传两种方式
- 实时显示上传进度（百分比、速度、剩余时间）
- 上传前进行文件大小和格式校验
- 上传成功后自动生成唯一文件标识（ID）
- 上传完成后自动生成分享链接和二维码

**限制条件：**
- 单文件大小上限：**2GB（暂时保持）**；前端需在上传区域明确展示"单文件 2GB 上限"
- 单次批量上传数量：**可配置**（默认建议不限/较大，但需考虑主Pad性能与稳定性）
- 支持的文件格式：默认不限（可配置黑名单，如可执行文件等）
- 文件名称长度限制：最多 255 字符

**异常处理：**
- 上传失败时显示明确的错误提示
- 网络中断：至少保证"失败可重试"；是否做断点续传需结合现有上传实现评估（可作为 P1）
- 文件重名：不覆盖原文件；后台以唯一 ID/文件名落盘，列表展示原始文件名即可

#### 2.2 文件列表管理

**需求描述：**
提供文件列表页面，展示所有已上传的文件及其详细信息。

**展示字段：**
- 文件名称（支持点击预览/下载）
- 文件大小
- 上传时间
- 上传者（用户名/IP 地址）
- 下载次数（显示当前下载次数）
- **有效期剩余天数**（按自然日计算，凌晨更新；例如"剩余 25 天"）
- 操作按钮（下载、生成二维码、删除、续期）

**交互功能：**
- 支持按文件名、上传者、上传时间搜索和筛选
- 支持按各字段排序
- 支持分页显示（默认每页 20 条）
- 支持批量删除

#### 2.3 文件下载功能

**需求描述：**
提供多种文件下载方式，方便用户获取文件。

**下载方式：**
1. **直接下载：** 在文件列表中点击下载按钮
2. **链接下载：** 复制分享链接在浏览器中打开下载（需要账密）
3. **二维码下载：** 扫描二维码进入下载页面（需要账密）

**功能要点：**
- 支持断点续传下载（依赖现有服务端 Partial Content，默认支持 Range）
- 下载页（面向移动端）需要尽量轻量：展示文件信息 + 下载按钮
- 下载前预览文件信息（文件名、大小、上传者、上传时间）
- 下载页面支持多终端适配（PC、手机、平板）

**下载页面内容：**
- **账密鉴权表单**：输入账户 `eastai` 和密码 `888888` 后才能查看文件详情与下载
- 文件基本信息（文件名、大小、上传时间、剩余有效天数）
- 下载按钮
- 有效期提示（例如"此文件保存 30 天，剩余 25 天"）
- 下载者信息采集（可选）：允许填写"姓名/备注"（用于满足"谁下载"可追溯；不强制）
- **当前 IP 今日下载次数显示**：例如"您今日已下载 5 次，剩余 95 次"

**鉴权与风控规则：**
- **下载账密**：统一使用账户 `eastai` + 密码 `888888`，所有分享链接共用此账密（便于记忆与传播）
- **分享链接格式**：`http://<主PadIP>:3000/t/<fileId>` 或 `http://eastaifile.com:3000/t/<fileId>`（fileId 为文件唯一标识，非随机 token）
- **频控（P0，必须实现）**：
  - 每个 IP 每天最多下载 **100 次**（跨所有文件累计）
  - 超过限制时返回明确提示："您今日下载次数已达上限（100次），请明天再试"
  - 在下载页面实时显示"今日已用次数/剩余次数"
  - 频控计数器每日凌晨重置
- **异常访问记录**：记录所有访问日志（IP、UA、文件ID、时间、结果），用于排查滥用

#### 2.4 操作记录追溯

**需求描述：**
完整记录文件的上传和下载操作，支持操作审计和问题追溯。

**记录内容：**

**上传记录：**
- 上传者信息（当前登录账号：固定 `admin` + 来源 IP + User-Agent）
- 上传时间（精确到秒）
- 文件信息（文件名、大小；是否计算 MD5 需评估成本，默认可不做或作为 P1）
- 操作状态（成功/失败）

**下载记录：**
- 下载者信息（来源 IP + User-Agent + 可选"姓名/备注"）
- 下载时间（精确到秒）
- 文件信息
- 下载完成状态

**查看方式：**
- 在文件列表中点击查看详情
- 提供独立的操作日志页面（或详情抽屉/弹窗）
- 支持导出操作记录（CSV；Excel 可作为 P1）
- 支持按时间范围、操作人、文件名筛选

#### 2.5 文件生命周期管理

**需求描述：**
自动管理文件的存储周期，避免无效文件长期占用存储空间。

**有效期规则：**
- 默认有效期：30 天（固定）
- 支持上传时指定单个文件的有效期（可选功能，P1）
- 支持手动续期（延长 7 天、30 天、永久，P1）

**过期提醒（简化版）：**
- **固定有效期**：所有文件保存 **30 天**（按自然日计算，从上传当天凌晨开始计算第 1 天）
- **剩余天数展示**：文件列表与下载页面实时显示"剩余 X 天"
- **即将过期标识**：剩余 ≤ 7 天时，在列表中显示"即将过期"标签（橙色）
- **过期时间计算**：按自然日，每日凌晨更新剩余天数；到期当天凌晨执行自动清理

**自动清理：**
- 定时清理策略：主Pad本地服务每日凌晨扫描过期文件并删除
- 边界情况：若主Pad服务长期不运行，需在**下次启动时补偿清理**
- 删除保留策略：默认直接删除（满足需求的"自动删除"）；如需"回收站"可作为 P1
- 删除需记录日志（删除人/删除方式：自动/手动/过期）

**存储与空间提醒（P0）：**
- **展示位置**：在**监控页面**（系统监控面板）中新增"文件中转存储"统计区域
- **展示内容**：
  - 当前文件中转目录占用空间（例如"已用 1.2 GB"）
  - 主Pad 磁盘剩余空间（例如"剩余 25.8 GB"）
  - 文件数量（例如"当前 52 个文件"）
- **阈值提醒**：例如占用超过 10 GB 或磁盘剩余低于 5 GB，在监控页与文件中转页面同时显示告警提示
- **硬阈值处理**：达到上限时禁止继续上传，并给出明确提示（避免主Pad磁盘被写爆影响系统稳定）

**手动删除：**
- 上传者可以随时删除自己上传的文件
- 管理员可以删除任何文件
- 删除前需要二次确认

#### 2.6 二维码分享功能

**需求描述：**
生成文件下载二维码，方便移动设备快速访问。

**功能要点：**
- 每个文件自动生成唯一的下载二维码
- **二维码内容**：`http://<主PadIP>:3000/t/<fileId>` 或 `http://eastaifile.com:3000/t/<fileId>`
- 支持下载二维码图片（PNG 格式）
- 二维码页面显示文件基本信息
- 扫码后跳转到下载页面，需要输入账密 `eastai/888888` 后才能下载

**二维码参数：**
- 尺寸：300x300 像素
- 容错率：M 级（15%）
- 包含内容：分享链接（例如 `http://eastaifile.com:3000/t/abc123`）

---

#### 2.7 与现有"素材库/播放"能力的边界（避免做重复功能）

- **素材库（Media）**：用于上传图片/视频并编排播放列表，终端播放会直接拉取 `/uploads/*` 资源。
- **本需求（文件中转/传文件）**：面向"临时文件分享/交付/拷贝"，文件类型不限，需要**下载审计**与**30 天过期清理**，且需要**账密鉴权的分享页/二维码下载**。
- 两者存储可共用同一落盘目录，但必须在数据模型与 API 路由层面**隔离**，避免：
  - 传文件被播放系统误引用
  - 传文件分享绕过审计/过期控制

- **局域网默认 HTTP**（现有架构为 `http://<ip>:3000`），安全控制以"管理页密码 + 下载统一账密"实现

#### 3.3 兼容性要求

- 浏览器支持：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 移动端适配：iOS 12+、Android 8+
- 屏幕分辨率：1920x1080 及以上（PC），375x667 及以上（移动）

#### 3.4 可用性要求

- 系统可用性 ≥ 99%
- 支持 7×24 小时运行
- 文件存储支持自动扩容
- 数据库支持定期备份（每日一次）

### 5. 基于现有项目的落地方案（对齐 Xplay 现状）

#### 5.1 现状复用点（无需新建独立后端）

- **管理端**：`apps/web-admin`（React + antd + axios），请求前缀 `/api/v1`。
- **服务端**：主Pad Android 内置本地服务 `LocalServerService`（Ktor/Netty，端口 3000）。
- **存储**：Room（SQLite）+ 本地文件落盘（`filesDir/uploads`）。

#### 5.2 建议的接口形态（便于做"账密鉴权分享 + 审计 + 过期"）

为避免绕过审计与过期控制，本功能的下载应**走专用分享接口/独立下载页**，不要直接暴露 `/uploads/{filename}` 作为分享链接。

**管理端接口（需要 admin 登录）：**
- **上传**：`POST /api/v1/transfer/upload`（multipart/form-data: file）
- **列表**：`GET /api/v1/transfer/files`（返回所有文件列表，包含剩余天数）
- **删除**：`DELETE /api/v1/transfer/files/{id}`
- **获取分享链接**：`GET /api/v1/transfer/files/{id}/share` → 返回 shareUrl + qrContent
- **日志查询**：`GET /api/v1/transfer/files/{id}/logs`

**分享/下载接口（需要账密鉴权 eastai/888888）：**
- **分享页**：`GET /t/{fileId}` → 展示登录表单，验证账密后显示文件信息与下载按钮
- **验证账密**：`POST /t/{fileId}/auth`（body: {username, password}）→ 验证 eastai/888888，成功后返回临时访问凭证
- **实际下载**：`GET /t/{fileId}/download`（携带临时凭证）→ 返回附件下载，并写入"下载日志"
- **频控检查**：每次访问 `/t/{fileId}` 或 `/t/{fileId}/download` 时，检查来源 IP 的今日下载次数，超过 100 次则拒绝并提示

说明：
- `/t/*` 路由不走 `/api/v1`，在现有鉴权拦截逻辑下可天然不要求 admin 登录
- **采用统一账密鉴权**：账户 `eastai` + 密码 `888888`，所有分享链接共用（便于记忆与传播）
- **频控**：每个 IP 每天累计下载 100 次上限，每日凌晨重置计数器

#### 5.3 数据模型建议（对齐 Room / SQLite）

- **transfer_files**：id、originalName、storedFilename、size、createdAt、expiresAt（按自然日，上传日 + 30天凌晨）、uploaderIp、uploaderUserAgent、downloadCount、status
- **transfer_logs**：id、fileId、action(upload/download/delete/expire_delete)、ip、userAgent、clientRemark(可选)、authResult(账密验证结果)、time、result
- **transfer_ip_quota**：ip、date（自然日，例如 2026-01-21）、downloadCount（当日累计下载次数，每日凌晨清零）

#### 5.4 过期清理与频控实现约束（Android 场景必须写清）

**过期清理：**
- **清理触发**：服务启动时补偿清理 + 每日凌晨定时清理
- **清理范围**：删除文件落盘 + 删除/更新 DB 状态 + 写入清理日志
- **过期文件访问**：分享页提示"文件已过期"，下载接口返回 410 状态
- **时间计算**：按自然日，上传当天为第 1 天，30 天后的凌晨执行清理

**频控实现：**
- **计数器**：`transfer_ip_quota` 表记录每个 IP 的每日下载次数
- **重置时机**：每日凌晨清理过期文件时，同步清空所有 IP 的计数器（删除旧日期记录）
- **检查逻辑**：每次下载前，查询 `transfer_ip_quota` 中当前 IP 今日计数，若 ≥ 100 则拒绝
- **显示逻辑**：在下载页面实时查询并显示"今日已下载 X 次，剩余 Y 次"
