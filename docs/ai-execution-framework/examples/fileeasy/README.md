# FileEasy Framework Onboarding Example

This example is intentionally limited to project registration.

It is not a feature task pack and does not dispatch implementation work.

Why:

- the framework should stay generic
- FileEasy should first prove document readiness
- business implementation tasks should only be created after the bundle is approved

Current signal from this example:

- PRD is registered
- workflow is registered
- design is registered but still marked as `candidate`

That means:

- structural bundle validation should pass
- strict ready validation should block implementation dispatch until design is promoted
