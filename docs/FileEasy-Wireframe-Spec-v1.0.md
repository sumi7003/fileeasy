## 2. 文档目标

本说明稿用于把视觉交互规范继续拆解成页面骨架定义，明确每个页面的模块顺序、内容层级、按钮位置、状态切换和端内差异，供 UI 出线框、前端搭页面和产品验收时共同使用。

本稿不定义高保真样式，不定义具体接口，仅定义“页面应当如何排布和运作”。

## 3. 页面清单

本次线框说明覆盖以下页面：

1. `A01` APK 首页
2. `W01` 扫码上传页登录前
3. `W02` 扫码上传页登录后
4. `W03` 上传任务状态变化
5. `W04` 上传页异常状态

说明：按 `FE-CR001`，FileEasy v1 不再覆盖远程 `/admin` 管理页线框。

## 4. 线框约定

### 4.1 区块命名

- `Header`：页面头部
- `Status`：状态区
- `Card`：主信息卡
- `Action`：操作区
- `List`：列表区
- `Modal`：弹窗层
- `Drawer`：抽屉层
- `Toast`：轻提示

### 4.2 优先级标记

- `P0`：必须首屏可见
- `P1`：重要，但可随页面下滚出现
- `P2`：辅助信息

### 4.3 线框表达原则

- 默认按手机优先描述
- 若桌面端结构不同，会单独补充
- 所有线框均以“从上到下、从左到右”的阅读顺序描述

## 5. `A01` APK 首页

### 5.1 页面目标

让管理员打开 APK 后，立即知道服务状态、上传入口和本机密码修改入口。

### 5.2 页面骨架

```text
+--------------------------------------------------+
| Header                                           |
| FileEasy                                         |
| 把这台设备作为局域网文件接收节点                 |
+--------------------------------------------------+
| Status                                           |
| [服务已开启]                                     |
+--------------------------------------------------+
| Main Card                                        |
| 上传二维码                                       |
| [                 QR CODE                      ] |
| 上传页地址                                       |
| http://192.168.1.23:3000/                        |
+--------------------------------------------------+
| Action Card                                      |
| 修改密码                                         |
| 更新上传页密码                                   |
+--------------------------------------------------+
```

### 5.3 区块定义

1. `Header` `P1`
   - 产品名
   - 一句简短说明
   - 不出现技术参数和复杂说明

2. `Status` `P0`
   - 状态标签
   - 状态辅助文案可选
   - 位置在二维码区上方

3. `Main Card` `P0`
   - 标题：上传二维码
   - 二维码主容器
   - 地址标题：上传页地址
   - 地址正文

4. `Action Card` `P1`
   - 主文案：修改密码
   - 辅助文案：更新上传页密码

### 5.4 状态变化

- 正常态：显示状态、二维码、地址、修改密码操作卡
- 服务启动中：状态区变为加载文案，二维码区可保留占位
- 无可用地址：二维码区域替换为空提示卡，地址显示为不可用
- 服务异常：状态区显示错误提示，二维码区不展示可扫码内容

### 5.5 无可用地址线框

```text
+--------------------------------------------------+
| Status                                           |
| [未检测到可用局域网地址]                         |
+--------------------------------------------------+
| Main Card                                        |
| 当前无法生成上传二维码                           |
| 请确认设备已连接 Wi-Fi 或热点                    |
+--------------------------------------------------+
```

### 5.6 端内适配

- 手机竖屏：所有区块单列堆叠
- 平板横屏：Header 与 Status 保持顶部，Main Card 置中，Action Card 放底部
- 电视盒子远距场景：优先放大二维码与地址字号，压缩辅助文案

## 6. `W01` 扫码上传页登录前

### 6.1 页面目标

用户进入页面后，先理解这是上传页，再完成密码验证。

### 6.2 页面骨架

```text
+--------------------------------------------------+
| Header                                           |
| FileEasy 文件上传                                |
| 上传到当前局域网设备                             |
+--------------------------------------------------+
| Upload Area (disabled)                           |
| [ 选择文件 ]                                     |
| 支持单文件 / 多文件上传                          |
| 单文件最大 4GB，支持断点续传                     |
+--------------------------------------------------+
| Login Card                                       |
| 输入密码后开始上传文件                           |
| [ 密码输入框                    ][显示/隐藏]     |
| [ 登录 ]                                         |
+--------------------------------------------------+
| Empty Task List                                  |
| 上传任务将在登录后显示                           |
+--------------------------------------------------+
```

### 6.3 区块定义

1. `Header` `P1`
   - 页面标题
   - 一句场景说明

2. `Upload Area` `P0`
   - 保持可见
   - 但为禁用态或半遮罩态
   - 让用户提前理解页面用途

3. `Login Card` `P0`
   - 密码输入框
   - 显隐切换
   - `登录` 主按钮

4. `Empty Task List` `P2`
   - 仅作占位说明
   - 不展示真实任务列表

### 6.4 密码错误态

```text
+--------------------------------------------------+
| Login Card                                       |
| 输入密码后开始上传文件                           |
| [ 密码输入框                    ][显示/隐藏]     |
| 密码错误，请重新输入                             |
| [ 登录 ]                                         |
+--------------------------------------------------+
```

### 6.5 桌面端建议

- 页面采用上下结构即可，不必强制分栏
- 登录卡片宽度应收敛，避免横向过长
- 上传区和登录区之间留出明确间距

## 7. `W02` 扫码上传页登录后

### 7.1 页面目标

让用户快速选择文件并持续看到上传进展。

### 7.2 手机端骨架

```text
+--------------------------------------------------+
| Header                                           |
| FileEasy 文件上传                                |
| 已连接到当前设备                                 |
+--------------------------------------------------+
| Upload Card                                      |
| [ 选择文件 ]                                     |
| 支持单文件 / 多文件上传                          |
| 单文件最大 4GB，支持断点续传                     |
+--------------------------------------------------+
| Task List                                        |
| 文件 A.pdf                 25%                   |
| [███████-----------------]                        |
| 上传中                                           |
|                                                  |
| 文件 B.zip                 等待上传              |
| [------------------------]                        |
+--------------------------------------------------+
```

### 7.3 桌面端骨架

```text
+------------------------------+-------------------+
| Header                                           |
| FileEasy 文件上传                                |
+------------------------------+-------------------+
| Upload Card                  | Task List         |
| [ 选择文件 ]                 | 文件 A.pdf  25%   |
| 支持单/多文件上传            | [进度条]          |
| 单文件最大 4GB               | 上传中            |
| 支持断点续传                 |                   |
|                              | 文件 B.zip 等待中 |
+------------------------------+-------------------+
```

### 7.4 区块定义

1. `Header` `P1`
   - 页面标题
   - 连接状态辅助文案

2. `Upload Card` `P0`
   - `选择文件` 主按钮
   - 支持能力说明
   - 不堆过多格式说明

3. `Task List` `P0`
   - 每项包含文件名、进度条、百分比、状态
   - 多文件时按任务顺序排列

### 7.5 文件任务单元

每个任务单元固定包含：

- 文件名
- 文件大小
- 进度值
- 状态文案
- 进度条

建议顺序：

1. 第一行：文件名 + 百分比或状态
2. 第二行：进度条
3. 第三行：文件大小 + 状态补充说明

## 8. `W03` 上传任务状态变化

### 8.1 状态集合

- 等待上传
- 上传中
- 上传中断
- 正在恢复
- 上传完成
- 上传失败

### 8.2 单任务状态线框

`等待上传`

```text
文件 A.mp4                     等待上传
[----------------------------]
2.1 GB
```

`上传中`

```text
文件 A.mp4                     42%
[████████████----------------]

```

`正在恢复`

```text
文件 A.mp4                     正在恢复
[████████████----------------]
2.1 GB | 正在恢复上传任务
```

`上传完成`

```text
文件 A.mp4                     上传完成
[████████████████████████████]
2.1 GB
```

`上传失败`

```text
文件 A.mp4                     上传失败
[█████████-------------------]
设备空间不足，请联系管理员清理
```

## 9. `W04` 上传页异常状态

### 9.1 空间不足

```text
+--------------------------------------------------+
| Alert Card                                       |
| 设备空间不足，请联系管理员清理                   |
| 当前无法发起新的上传                             |
+--------------------------------------------------+
```

处理规则：

- Alert Card 出现在 Upload Card 上方
- `选择文件` 按钮禁用
- 任务列表仍可见

### 9.2 网络中断

```text
+--------------------------------------------------+
| Inline Notice                                    |
| 网络中断，正在等待恢复                           |
+--------------------------------------------------+
```

处理规则：

- 显示在任务列表上方或对应任务项内
- 不清空现有进度
- 待网络恢复后进入“正在恢复”

### 9.3 续传状态过期

```text
+--------------------------------------------------+
| Inline Notice                                    |
| 上传任务已过期，请重新上传                       |
+--------------------------------------------------+
```

处理规则：

- 对应任务项标为失败
- 不再展示可继续恢复的暗示

## 10. 远程管理页移出范围

按 `FE-CR001`，以下线框不再属于 FileEasy v1：

- 远程 `/admin` 管理页
- 远程文件列表
- 远程文件预览层
- 远程重命名弹窗
- 远程删除确认
- 远程批量删除模式
- 远程下载入口

如果后续恢复文件管理能力，应作为 v1.1 或独立需求重新产出线框，不沿用本 v1 线框说明直接开发。

## 11. 页面流转说明

### 11.1 主流程

1. 管理员打开 APK 首页
2. 上传用户扫码进入上传页
3. 用户输入密码登录
4. 用户选择单文件或多文件
5. 用户查看上传进度与结果

### 11.2 上传页状态流转

```text
登录前
  -> 登录成功
登录后
  -> 选择文件
任务创建
  -> 上传中
上传中
  -> 上传完成
上传中
  -> 网络中断
网络中断
  -> 正在恢复
正在恢复
  -> 上传完成 / 上传失败
```

### 11.3 `/admin` 访问流转

```text
访问 /admin
  -> 404 / 410 / 返回上传页
  -> 不展示远程文件管理能力
```
